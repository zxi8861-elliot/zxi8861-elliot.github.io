# Hsin Yael 个人作品浏览器界面思维导图

> 目标：不要做成传统单页滚动网站，而是参考 Heyefi 截图中的固定分栏作品浏览界面。  
> 核心关键词：固定分栏、作品浏览器、像素字体、设计档案、项目预览、右侧信息栏、非传统长页面。

## 一句话方向

请不要把它做成传统单页滚动网站。我要的是类似设计师 portfolio 操作界面的固定分栏布局：左侧固定导航，中间是介绍和项目列表，右侧是项目大图预览，最右是联系和项目元信息。页面像一个全屏作品浏览器，而不是 Hero / About / Contact 纵向堆叠页面。

## 网站定位

- 个人作品浏览器
- 视觉型 Portfolio
- 桌面端优先
- 设计师 / AI 产品经理个人主页
- 非传统滚动长页
- 像一个自定义设计系统界面，而不是普通个人介绍页

### 全屏固定画布

- 页面高度接近 `100vh`
- 横向分栏布局
- 主要内容不按普通区块纵向堆叠
- 页面整体像一个固定的桌面应用界面
- 滚动只发生在局部内容区，例如项目详情或移动端页面

## 页面状态

### 默认 Work 状态

- 左侧导航中 `Work` 高亮
- 中左区域展示个人主标题和项目列表
- 中右区域展示第一个项目的大图
- 右侧信息栏展示第一个项目的元信息

### 项目选中状态

- 被选中的项目行高亮
- 中右大图切换
- 右侧项目信息切换
- 中左区域下方展开项目说明：
  - `About`
  - `Challenge`
  - `Solution`
  - `Project Scope`
- 底部状态栏显示：
  - `C:\HSINYAEL\WORK`
  - `PROJECT 001 / 005`
  - `STATUS VIEWING`

### About 状态

- 左侧导航中 `About` 高亮
- 保留左侧导航和右侧联系栏
- 中央切换为个人介绍
- 右侧视觉区变为个人照片、身份卡片或蓝底 ID 卡
- 内容重点：
  - 我是谁
  - 正在关注什么
  - 做过哪些类型的 AI 产品
  - 我的工作方式
  - 如何联系我

### Playground 状态

- 左侧导航中 `Playground` 高亮
- 展示实验项目、概念原型、AI 小工具
- 可以使用网格或列表
- 仍然保持固定分栏界面，不变成普通长页面

### Contact 状态

- 可以不单独做成传统联系区
- 点击 `SAY HEY` 或 `Contact` 后：
  - 右侧栏强化显示邮箱
  - 或弹出极简联系面板
  - 或复制邮箱

## 视觉语言

### 字体

- 像素感 / 等宽字体
- 大标题强烈、粗粝、直接
- 小字偏终端风格
- 项目编号和元信息使用 monospace
- 避免温柔、圆润、SaaS 模板感字体

### 色彩

- 背景：白色 / 浅灰 / 米白
- 主文字：黑色
- 次级文字：灰色
- 强调色：电光蓝
- 状态点：绿色
- 视觉丰富度主要来自项目图片，而不是大面积渐变

推荐色值：

```css
background: #f4f4f1;
foreground: #111111;
muted: #e8e8e6;
mutedForeground: #7a7a76;
border: #d8d8d4;
accent: #0500ff;
accentForeground: #ffffff;
signal: #25e85c;
```

### 分割线与界面感

- 使用细灰线划分栏位
- 项目列表每行有横线
- 右侧信息栏像 inspector panel
- 页面底部可以有系统路径 / 状态栏
- 整体感觉像设计软件、文件系统或作品档案界面

### 图像风格

- 大幅项目海报
- 产品截图拼贴
- Polaroid / ID 卡片感
- 局部大图裁切
- 图片上可叠加线框、文字、编号、标签
- 可以有轻微颗粒感或扫描质感
- 避免普通头像墙、普通卡片封面、普通 banner 图

## 交互方式

- 点击导航切换内容状态
- 点击项目切换项目详情
- 项目行 hover 高亮
- 当前项目有选中背景
- 鼠标移动时可显示蓝色坐标小标签
- 右侧链接 hover 下划线
- `SAY HEY` 按钮 hover 时有高对比变化
- 移动端从横向分栏改为纵向压缩版

## 响应式规则

### 桌面端

- 保持四栏布局：
  - 左侧导航栏
  - 中左文字和项目列表
  - 中右大图预览
  - 右侧联系和元信息栏
- 视觉重点在项目大图
- 左右侧栏固定

### 平板端

- 左侧导航变窄
- 右侧信息栏可以合并到项目视觉下方
- 项目列表和视觉展示保持并排

### 移动端

- 不强行保留四栏
- 顶部显示 Logo 和导航
- 先显示大标题
- 再显示当前项目视觉
- 再显示项目列表和项目详情
- 联系方式放在页面底部或抽屉中
- 仍然保留像素字体、状态栏、编号、元信息等视觉 DNA
