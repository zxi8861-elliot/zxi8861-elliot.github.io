# PRD：地理自然环境音 App

## 1. 产品概述

### 1.1 产品名称

暂定名：**听见此地 / SoundMap / 地声**

### 1.2 产品一句话描述

一款基于地图浏览的自然与环境声音 App，用户可以通过中国地图探索不同地区的自然音、城市环境音与文化场景音，用于助眠、放松、专注和声音旅行。

### 1.3 产品定位

本产品不是传统音乐播放器，也不是纯白噪音工具，而是一个以「地点」为入口的声音探索 App。

用户不仅可以听雨声、海浪、森林、虫鸣等常规白噪音，还可以通过地图点击不同地区，收听来自该地区的自然环境声、城市环境声和特色文化声音。

### 1.4 一期核心目标

一期目标是做出一个可在国内合规上架、功能闭环完整、个人开发者可实现的 MVP。

一期重点不做全球电台、不做用户上传、不做复杂社区，优先实现：

1. 中国地图声音浏览
2. 音效详情页
3. 音频播放、循环、收藏、历史
4. 睡眠定时关闭
5. 基础搜索与分类
6. 后台播放
7. 音源版权可追溯

### 1.5 国内上架合规前提

在中国大陆提供联网 App 服务，需要关注 App 备案、隐私政策、用户协议、版权授权、内容安全等问题。

如果 App 包含用户生成内容，例如用户上传音频、评论、公开分享等，还需要更强的内容审核、举报、屏蔽与未成年人保护机制。

因此，一期建议采用 **平台自有音源 / 开源授权音源 / 自采音源**，暂不开放用户上传，降低审核与运营风险。

## 2. 用户与场景

### 2.1 目标用户

#### 核心用户

1. 有助眠需求的人  
   使用自然音、白噪音、环境音帮助入睡。

2. 有专注需求的人  
   在学习、工作、阅读时播放稳定的环境音。

3. 对地理、旅行、城市声音感兴趣的人  
   想通过声音探索某个城市、海边、森林、沙漠或乡村。

4. 喜欢 ASMR / 白噪音 / 冥想音频的人  
   但希望内容比普通白噪音 App 更有地域感和探索感。

### 2.2 典型用户场景

#### 场景 1：睡前听自然音

用户晚上准备睡觉，打开 App，点击「海边」分类，选择「东海海岸浪声」，设置 30 分钟定时关闭和渐弱淡出，然后锁屏入睡。

#### 场景 2：通过地图探索声音

用户打开中国地图，点击云南区域，看到「雨林」「溪流」「村寨清晨」「虫鸣」等声音列表，点击其中一个播放。

#### 场景 3：专注工作

用户工作时搜索「咖啡馆」或「雨声」，选择一个平稳音效，开启循环播放，并加入收藏。

#### 场景 4：声音旅行

用户对沙漠感兴趣，点击西北区域，收听「沙漠风声」「戈壁夜晚」「驼铃远声」等声音，并查看声音来源说明。

#### 场景 5：回听常用声音

用户每天睡觉前都听同一段雨声，可以在收藏或播放历史中快速找到。

## 3. 产品范围

### 3.1 一期 MVP 范围

一期必须完成：

| 模块 | 是否一期 | 说明 |
|---|---:|---|
| 中国地图声音浏览 | 是 | 产品差异化核心 |
| 声音分类浏览 | 是 | 自然音、城市音、文化音 |
| 音频播放 | 是 | 播放、暂停、停止、循环 |
| 后台播放 | 是 | 助眠场景必需 |
| 定时关闭 | 是 | 5/10/30/60 分钟、自定义 |
| 渐弱关闭 | 是 | 提升助眠体验 |
| 收藏 | 是 | 用户留存基础 |
| 播放历史 | 是 | 便于回听 |
| 搜索 | 是 | 按标题、标签、地点搜索 |
| 音源说明 | 是 | 合规与信任基础 |
| 隐私政策 / 用户协议 | 是 | 上架必需 |
| App 备案信息展示位 | 是 | 国内上架准备 |
| 全球电台 | 否 | 合规风险较高，后置 |
| 用户上传声音 | 否 | UGC 风险高，后置 |
| 评论社区 | 否 | 内容审核成本高，后置 |
| 账号系统 | 可选 | 一期可先本地化，降低复杂度 |

### 3.2 一期不做

1. 不做全球实时电台。
2. 不做用户上传音频。
3. 不做公开评论、弹幕、社区。
4. 不做 AI 生成声音。
5. 不做复杂会员体系。
6. 不做跨端同步。
7. 不做实时定位自动推荐。
8. 不做全球地图全量音源。

### 3.3 二期可扩展

1. 全球地图声音探索。
2. 全球公开电台聚合。
3. 城市实时环境音。
4. 用户投稿音源，但需人工审核。
5. 主题声音专辑。
6. 睡眠报告。
7. 冥想 / 呼吸引导。
8. 声音混音功能。
9. 付费订阅。
10. 小组件 / 桌面组件。

## 6. 核心功能需求

## 6.1 首页

### 6.1.1 页面目标

让用户快速进入常用声音、睡眠场景和地图探索。

### 6.1.2 页面模块

#### A. 顶部问候区

展示文案：

- 晚上好，想听见哪里？
- 今天想用什么声音陪你？
- 选择一段声音，慢慢安静下来。

#### B. 搜索入口

支持搜索：

- 声音名称
- 地点
- 标签
- 分类

示例：

- 雨声
- 海浪
- 云南
- 森林
- 咖啡馆
- 火车站

#### C. 今日推荐声音

展示 3-5 个推荐音效。

字段：

```json
{
  "id": "sound_001",
  "title": "东海岸边的夜浪",
  "location": "浙江 · 舟山",
  "category": "自然音",
  "duration": 1800,
  "cover": "image_url",
  "tags": ["海浪", "夜晚", "助眠"]
}
```

#### D. 睡前推荐

展示适合睡眠的音效。

分类可包括：

- 轻雨
- 海浪
- 森林夜晚
- 溪流
- 风声
- 白噪音

#### E. 地图探索入口

展示一个中国地图卡片，点击进入地图页。

#### F. 最近播放

如果用户有播放历史，则展示最近 3 条。

### 6.1.3 交互需求

1. 点击声音卡片，进入声音详情页并可立即播放。
2. 点击地图卡片，进入地图页。
3. 点击搜索框，进入搜索页。
4. 点击最近播放，恢复该音频播放。

## 6.2 地图页

### 6.2.1 页面目标

让用户通过地理位置发现声音，形成产品差异化。

### 6.2.2 地图范围

一期仅做中国地图。

建议按以下层级实现：

1. 全国地图
2. 省级区域
3. 城市/地点声音列表

一期可以不做复杂 GIS，只需要可点击的中国省份 SVG 地图或简化地图。

### 6.2.3 地图展示规则

每个省份根据是否有音源展示不同状态：

| 状态 | 说明 |
|---|---|
| 有音源 | 可点击，高亮 |
| 暂无音源 | 可点击或不可点击，弱化 |
| 推荐地区 | 可特殊标记 |
| 当前选中 | 高亮 |

### 6.2.4 点击省份后展示内容

点击省份后，底部弹出声音列表。

字段：

```json
{
  "province": "云南",
  "description": "来自雨林、溪流、山地村落的自然声音。",
  "soundCount": 12,
  "sounds": [
    {
      "id": "sound_101",
      "title": "西双版纳雨林清晨",
      "category": "自然音",
      "duration": 1200,
      "tags": ["雨林", "鸟鸣", "清晨"]
    }
  ]
}
```

### 6.2.5 地图页交互

1. 用户可缩放地图。
2. 用户可点击省份。
3. 用户可查看该地区声音列表。
4. 用户可点击声音进入详情页。
5. 如果该地区暂无音源，展示空状态：
   > 这里的声音还在采集中，先听听附近的声音吧。

### 6.2.6 MVP 简化方案

如果地图开发成本较高，一期可采用：

- 中国地图 SVG
- 省份点击事件
- 不做精确经纬度
- 声音数据绑定 province 字段
- 后续再扩展城市级地图

### 6.4.5 合规要求

1. 每条音源必须有来源字段。
2. 每条音源必须有授权字段。
3. 不允许使用来源不明的音乐、影视、播客、广播录音。
4. 不允许使用包含明确个人对话、隐私信息、敏感内容的环境录音。
5. 如果音频中有人声，需要确认是否可公开传播。

## 6.5 播放器

### 6.5.1 基础播放能力

播放器必须支持：

1. 播放
2. 暂停
3. 停止
4. 进度条
5. 当前播放时间
6. 总时长
7. 循环播放
8. 上一首 / 下一首
9. 音量控制
10. 后台播放

### 6.5.2 播放模式

一期支持：

| 模式 | 说明 |
|---|---|
| 单曲播放 | 播放完停止 |
| 单曲循环 | 适合助眠 |
| 列表顺序播放 | 分类列表连续播放 |
| 随机播放 | 可后置 |

### 6.5.3 迷你播放器

当用户离开声音详情页时，底部显示迷你播放器。

迷你播放器展示：

1. 声音标题
2. 地点
3. 播放 / 暂停按钮
4. 定时关闭状态
5. 点击进入完整播放器

### 6.5.4 后台播放

要求：

1. App 进入后台后音频继续播放。
2. 锁屏页面显示当前音频信息。
3. 锁屏页面可暂停 / 继续。
4. 定时关闭在后台仍然生效。

## 7. 内容与音源策略

## 7.1 音源类型

一期可使用三类音源：

### A. 开源授权音源

来源可以是公开音频素材网站或开放数据集。

要求：

1. 明确授权协议。
2. 允许 App 内播放。
3. 允许商业或非商业使用，取决于产品未来商业化计划。
4. 可署名。
5. 可追溯原始链接。

### B. 自采音源

你自己或合作采集者用录音设备采集。

优点：

1. 版权最可控。
2. 更容易做地域特色。
3. 可形成产品独家内容。

要求：

1. 避免录入私人对话。
2. 避免录入受版权保护的音乐。
3. 记录采集地点、时间、设备、处理方式。
4. 音频中如有明显人声，谨慎使用。

### C. 合作授权音源

与声音创作者、自然录音师、地方文旅机构合作。

一期不必做，但二期可考虑。

## 7.2 不建议一期使用的音源

1. 影视剧、综艺、短视频中的音频。
2. 未授权音乐。
3. 广播电台实时流。
4. 播客片段。
5. 含私人对话的街头录音。
6. 来源不明的网盘素材。
7. AI 生成但授权不清的声音。

## 7.3 一期音源数量建议

建议一期先准备：

| 类型 | 数量 |
|---|---:|
| 自然音 | 40-60 条 |
| 城市环境音 | 20-30 条 |
| 人文场景音 | 10-20 条 |
| 睡眠专用精选 | 20 条 |
| 合计 | 70-110 条 |

最低 MVP 可从 30 条开始，但建议地图覆盖感要足够，因此最好至少覆盖 15-20 个省级区域。

## 7.4 音频处理标准

建议统一处理：

1. MP3 / AAC 格式。
2. 单条音频 5-30 分钟。
3. 睡眠音频避免突然大音量。
4. 做响度标准化。
5. 做基础降噪。
6. 文件大小控制在移动端可接受范围。
7. 可循环音频尽量处理首尾衔接。

## 7.5 音源元数据标准

每条音源必须有完整元数据：


## 8. 合规与上架策略

## 8.1 国内上架建议

一期建议按「普通音频播放工具 + 地图浏览 + 助眠放松」定位，而不是「电台」「社交」「UGC 社区」。

推荐应用分类：

- 健康健美 / 睡眠放松
- 生活
- 音乐与音频
- 工具

避免宣传为：

- 全球实时电台
- 实时监听世界
- 用户自由上传声音社区
- 未经审核的全球广播聚合

## 8.2 必备合规材料

建议准备：

1. App 备案。
2. 隐私政策。
3. 用户协议。
4. 第三方 SDK 清单。
5. 个人信息收集清单。
6. 软著，安卓渠道通常强烈建议甚至可能要求。
7. 音源版权说明。
8. App 图标、截图、介绍文案。
9. 后台播放权限说明。
10. 联系邮箱。

## 8.3 隐私合规建议

一期尽量少收集数据。

### 不登录模式

一期可以不做账号系统，降低个人信息收集。

### 可能收集的数据

| 数据 | 是否必要 | 用途 |
|---|---:|---|
| 设备信息 | 可选 | 崩溃分析 |
| 播放历史 | 是 | 本地功能 |
| 收藏记录 | 是 | 本地功能 |
| 搜索记录 | 可选 | 本地体验优化 |
| 精确定位 | 否 | 一期不需要 |
| 通讯录 | 否 | 禁止 |
| 麦克风 | 否 | 一期不需要 |
| 相册 | 否 | 一期不需要 |

### 定位权限

一期不建议申请定位权限。

地图是内容浏览入口，不需要用户真实位置。

## 8.4 电台功能风险

全球电台功能建议后置，原因：

1. 内容不可控。
2. 可能包含新闻、政治、宗教、成人、广告等内容。
3. 可能涉及境外信息源。
4. 版权授权不清。
5. 国内应用商店审核不确定性高。
6. 需要实时内容审核能力。

如果二期要做，建议先做：

- 白名单电台
- 国内合规电台
- 无新闻/无谈话内容的音乐或自然类电台
- 明确授权来源
- 可一键下架

## 8.5 UGC 功能风险

用户上传声音也建议后置。

如果未来做 UGC，需要：

1. 用户实名/账号体系。
2. 上传前协议确认。
3. 内容审核。
4. 版权声明。
5. 举报入口。
6. 拉黑/屏蔽。
7. 下架机制。
8. 未成年人保护。
9. 敏感内容识别。
10. 人工复审流程。

## 9. 技术方案建议

## 9.1 个人开发者推荐技术栈

### 方案 A：Flutter

适合目标：

- 同时做 iOS 和 Android
- 音频播放体验较好
- 地图 / SVG / 动画能力不错

推荐：

```markdown
前端：Flutter
状态管理：Riverpod / Provider
本地存储：Hive / SQLite
音频播放：just_audio
后台播放：audio_service
地图：中国 SVG 自定义组件
后端：Supabase / Firebase / 自建 Node.js
存储：对象存储 OSS / COS / R2
```

### 方案 B：React Native

适合目标：

- 你希望更容易和 Web coding / AI coding 配合
- JS/TS 生态更方便
- 后续可复用 Web 管理后台

推荐：

```markdown
前端：React Native / Expo
状态管理：Zustand
本地存储：AsyncStorage / SQLite
音频播放：expo-av 或 react-native-track-player
地图：中国 SVG / react-native-svg
后端：Supabase / Node.js
存储：阿里云 OSS / 腾讯云 COS / Cloudflare R2
```

### 方案 C：先做 Web App / H5 原型

适合目标：

- 快速验证想法
- 先用 AI coding 做出来
- 暂时不上架

推荐：

```markdown
前端：Next.js
UI：Tailwind CSS
状态管理：Zustand
音频播放：HTML5 Audio / Howler.js
地图：ECharts 中国地图 / SVG
数据：JSON / Supabase
存储：对象存储
```

### 推荐路线

你的身份是产品经理，想个人开发，建议：

```markdown
第一步：Next.js Web 原型
第二步：确认体验和数据结构
第三步：用 React Native 或 Flutter 做 App
第四步：准备国内上架材料
```

如果目标是一开始就上架，建议直接做 Flutter 或 React Native。

## 14. 风险分析

## 14.1 最大风险：音源版权

风险：

- 使用来源不明素材。
- 授权协议不支持商用。
- 忘记署名。
- 音频中包含音乐、广播、人声隐私。

解决方案：

1. 每条音源建立版权字段。
2. 保留原始来源链接。
3. 优先使用自采和明确 CC 授权音源。
4. 不使用来源不明素材。
5. 上线前人工审核音频。

## 14.2 国内上架风险

风险：

- App 未备案。
- 隐私政策不完整。
- 权限申请过多。
- 音源版权说明不足。
- 涉及电台、UGC、境外内容。

解决方案：

1. 一期不做电台。
2. 一期不做 UGC。
3. 一期不申请定位、麦克风、通讯录。
4. 做 App 备案。
5. 明确音源来源。
6. 准备隐私政策、用户协议、SDK 清单。

## 14.3 内容规模不足

风险：

- 地图看起来空。
- 用户点了省份没有声音。
- 体验不完整。

解决方案：

1. 先覆盖重点区域。
2. 每个区域至少 2-3 条声音。
3. 暂无声音区域展示推荐附近声音。
4. 首页重点推荐高质量声音，而不是暴露内容不足。

## 14.4 个人开发成本过高

风险：

- 地图、播放器、后台播放、上架合规同时做，工作量大。

解决方案：

1. 先做 Web 原型。
2. 地图先用 SVG 简化。
3. 收藏和历史先本地化。
4. 后台管理先用表格或 JSON。
5. 不做账号系统。
6. 不做 UGC。

## 15. 一期推荐 MVP 版本定义

### 15.1 MVP 必须有

```markdown
1. 首页
2. 中国地图页
3. 省份声音列表
4. 声音详情页
5. 播放器
6. 后台播放
7. 定时关闭
8. 渐弱关闭
9. 收藏
10. 播放历史
11. 分类浏览
12. 搜索
13. 音源来源说明
14. 用户协议
15. 隐私政策
```

### 15.2 MVP 可以没有

```markdown

2. 云同步
3. 用户上传
4. 评论
5. 全球电台
6. 全球地图
7. 会员付费
8. AI 生成声音
9. 混音
10. 睡眠报告
```

```markdown

```

## 17. 最终建议

这个产品是可行的，但一期一定要克制。

最推荐的一期方向是：

> **不是“全球声音平台”，而是“基于中国地图的助眠自然环境音 App”。**

这样有三个好处：

1. **产品差异化明确**：比普通白噪音 App 多了地理探索感。
2. **开发成本可控**：地图 + 播放器 + 本地收藏历史即可形成闭环。
3. **上架风险较低**：不碰 UGC、不碰全球电台、不碰实时内容，重点做版权清晰的自然音和环境音。

全球电台、用户上传、声音社区都可以成为未来版本，但不建议放进一期。
